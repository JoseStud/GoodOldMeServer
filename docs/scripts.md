# Scripts & Utilities

This document catalogs helper scripts in `scripts/` and CI automation scripts in `.github/scripts/`.

## Catalog

| Script | Purpose | Parameters (args/env) | Exit code behavior | Idempotency | Required tools/dependencies | Example |
|-------|---------|------------------------|--------------------|-------------|-----------------------------|---------|
| `scripts/cloudflare-dns.sh` | Create or update a Cloudflare DNS A record for `<stack>.<BASE_DOMAIN>`. | Args: `<STACK_NAME> <IP_ADDRESS>`. Env: `ZONE_ID`, `CLOUDFLARE_API_TOKEN`, `BASE_DOMAIN`. | `0` on successful create/update, `1` on usage/validation/API errors. | Conditionally idempotent (updates existing record, creates if missing). | `bash`, `curl`, `jq`, Cloudflare API access. | `./scripts/cloudflare-dns.sh portainer 192.168.1.50` |
| `scripts/portainer-webhook.sh` | Trigger one or more Portainer GitOps webhooks. | Args: one or more webhook URLs, or env `WEBHOOK_URLS` (CSV). | `0` when all calls return `200/204`; `1` if usage invalid or any webhook fails. | Not idempotent at workflow level (re-trigger requests), but safe for redeploy semantics. | `bash`, `curl`. | `WEBHOOK_URLS="https://.../uuid1,https://.../uuid2" ./scripts/portainer-webhook.sh` |
| `scripts/import_infisical_secrets.sh` | Seed placeholder secrets across Infisical paths. | Env: logged-in Infisical CLI context; fixed `ENV=prod` in script. | Uses `set -e` but each `infisical secrets set ... || true` suppresses per-key failures; exits non-zero only on unrecoverable shell errors. | Mostly idempotent for initial seed (re-runs overwrite existing values). | `bash`, `infisical` CLI auth/session. | `./scripts/import_infisical_secrets.sh` |
| `scripts/archive/portainer-deploy.sh` *(archived)* | Legacy REST update path for Portainer stack env vars. | Args: `<stack_name> <env_file_path>`. Env: `BASE_DOMAIN`, `PORTAINER_TOKEN`, optional `ENDPOINT_ID`. | `0` on HTTP `200/204`; `1` on validation/not-found/API failures. | Conditionally idempotent for same env payload. | `bash`, `curl`, `jq`, `grep`, `sed`. | `BASE_DOMAIN=example.com PORTAINER_TOKEN=... scripts/archive/portainer-deploy.sh gateway .env` |
| `scripts/archive/update_deploy.py` *(archived)* | One-time migration helper to add `update_config.order: start-first` across compose files. | No args. Hardcoded file list. | Always exits `0` unless Python/file I/O fails. | Idempotent after target files already contain desired config. | `python3`. | `python3 scripts/archive/update_deploy.py` |
| `.github/scripts/render_inventory_from_tfc_outputs.sh` | Build deterministic `inventory-ci.yml` from TFC outputs. | Args: `<workspace_name> [output_file]`. Env: `TFC_TOKEN`, `TFC_ORGANIZATION`, optional `TFC_API_URL`. | `0` on success; non-zero on missing env/outputs/API failures. | Idempotent for same workspace state/output path. | `bash`, `curl`, `jq`. | `TFC_TOKEN=... TFC_ORGANIZATION=my-org .github/scripts/render_inventory_from_tfc_outputs.sh goodoldme-infra` |
| `.github/scripts/trigger_webhooks_with_gates.sh` | Dependency-ordered webhook trigger with pre/post health gates from `stacks/stacks.yaml`. | Args: `<manifest_path> [changed_stacks_csv]` or env `STACKS_CSV`; env `WEBHOOK_URL_*`, optional `BASE_DOMAIN`. | `0` on successful ordered redeploy; `1` on missing stack, missing webhook, health timeout, dependency cycle, etc. | Partially idempotent (re-triggers redeploys; health checks are repeat-safe). | `bash`, `curl`, `yq`, manifest file. | `STACKS_CSV="gateway,auth" .github/scripts/trigger_webhooks_with_gates.sh stacks/stacks.yaml` |
| `.github/scripts/detect_iac_impact.sh` | Resolve IaC impact outputs for validation workflow. | Env-only contract (event context + path-filter booleans + git refs). | `0` on valid output write; `1` on missing required env or dependency failures. | Idempotent for same git/event inputs. | `bash`, `git`, `jq`. | Invoked by `.github/workflows/iac-validation.yml` |
| `.github/scripts/resolve_meta_context.sh` | Resolve meta-pipeline execution toggles from push/dispatch/manual contexts. | Env-only contract (event, inputs, payload, filter booleans). | `0` on valid context; `1` for invalid SHA format or missing required env. | Idempotent for same event payload and refs. | `bash`, `git`, `awk`. | Invoked by `meta-pipeline.yml` and `meta-pipeline-smoke.yml` |
| `.github/scripts/build_network_access_policy.sh` | Build canonical `network_access_policy` JSON from runner public IPs or break-glass override. | Env: optional `BREAK_GLASS_POLICY_JSON` / `BREAK_GLASS_POLICY_FILE`; optional `GITHUB_OUTPUT`. | `0` on valid policy; `1` on schema/family validation failures or dependency errors. | Idempotent for same override or same observed runner IPs. | `bash`, `curl`, `jq`, `python3`. | `BREAK_GLASS_POLICY_JSON='{"oci_ssh":{"enabled":true,"source_ranges":["203.0.113.10/32"]},"gcp_ssh":{"enabled":true,"source_ranges":["2001:db8::/128"]},"portainer_api":{"source_ranges":["203.0.113.10/32","2001:db8::/128"]}}' .github/scripts/build_network_access_policy.sh` |
| `.github/scripts/sync_network_access_policy.sh` | Atomically sync policy to Terraform Cloud var and Infisical allowlist secret, then verify read-after-write. | Env: `NETWORK_ACCESS_POLICY_JSON`, `TFC_TOKEN`, `TFC_ORGANIZATION`, `TFC_WORKSPACE_INFRA`, `INFISICAL_PROJECT_ID`. | `0` on full sync+verify success; `1` on API/set/get mismatches or missing env. | Idempotent for same policy JSON. | `bash`, `curl`, `jq`, `infisical` CLI auth/session. | `NETWORK_ACCESS_POLICY_JSON='...' .github/scripts/sync_network_access_policy.sh` |
| `.github/scripts/preflight_network_access.sh` | Fail-closed preflight for runner egress policy and optional SSH/Portainer reachability checks. | Env: `NETWORK_ACCESS_POLICY_JSON`; optional toggles `RUN_ANSIBLE`, `RUN_CONFIG`, `RUN_HEALTH`, `RUN_PORTAINER`, `PORTAINER_API_URL`, `INVENTORY_FILE`. | `0` when all required checks pass; `1` when policy mismatch or reachability checks fail. | Idempotent for same policy and network state. | `bash`, `curl`, `jq`, `python3`, `nc`. | `RUN_ANSIBLE=true INVENTORY_FILE=inventory-ci.yml NETWORK_ACCESS_POLICY_JSON='...' .github/scripts/preflight_network_access.sh` |
| `.github/scripts/wait_for_portainer_allowlist_propagation.sh` | Poll Portainer API until allowlist propagation permits current runner. | Env: `PORTAINER_API_URL`; optional `WAIT_TIMEOUT_SECONDS`, `POLL_INTERVAL_SECONDS`. | `0` on HTTP `200/401`; `1` on timeout or invalid input. | Idempotent polling behavior. | `bash`, `curl`. | `PORTAINER_API_URL=https://portainer-api.example.com .github/scripts/wait_for_portainer_allowlist_propagation.sh` |
| `.github/scripts/assert_tfc_workspace_local_mode.sh` | Enforce `operations=false` for local Terraform CLI apply workflow. | Env: `TFC_TOKEN`, `TFC_ORGANIZATION`, `TFC_WORKSPACE`. | `0` when workspace local mode is set; `1` otherwise. | Idempotent read-only assertion. | `bash`, `curl`, `jq`. | `TFC_TOKEN=... TFC_ORGANIZATION=my-org TFC_WORKSPACE=goodoldme-portainer .github/scripts/assert_tfc_workspace_local_mode.sh` |
| `.github/scripts/verify_trusted_stacks_sha.sh` | Verify requested stacks SHA ancestry and CI status against stacks repo. | Arg: `<STACKS_SHA>`. Env: `STACKS_REPO_READ_TOKEN` or `GITHUB_TOKEN`; optional owner/repo/branch/api vars. | `0` when trusted; `1` on invalid SHA, API failure, failed checks/status, or trust violations. | Idempotent read-only verification. | `bash`, `curl`, `jq`, GitHub API token with repo read/check access. | `.github/scripts/verify_trusted_stacks_sha.sh 0123456789abcdef0123456789abcdef01234567` |
| `.github/scripts/wait_for_tfc_run.sh` | Poll Terraform Cloud run until terminal success/failure/timeout. | Args: `<RUN_ID> <WORKSPACE_NAME>`. Env: `TFC_TOKEN`; optional `TFC_API_URL`, `WAIT_TIMEOUT_SECONDS`, `POLL_INTERVAL_SECONDS`, `TFC_ORGANIZATION`. | `0` on `planned_and_finished` or `applied`; `1` on failed terminal states or timeout. | Idempotent polling behavior. | `bash`, `curl`, `jq`. | `TFC_TOKEN=... .github/scripts/wait_for_tfc_run.sh run-abc123 goodoldme-infra` |

## Notes

- CI scripts are intended to run inside GitHub Actions jobs with explicit env contracts.
- Archived scripts are retained for historical reference and should not be used in the normal deployment path.
- For operational deployment flows, see [Deployment Runbook](deployment-runbook.md).
